Quick Start

Squid controls local coding-agent CLIs — it doesn't replace them. Install at least one backend first, then start Squid.

1. Install a backend CLI

Pick one to start. You can add more agents later, mixing backends in the same workspace.

BackendCLIInstall
Claude Codeclaudecurl -fsSL https://claude.ai/install.sh | bash
OpenAI Codexcodexcurl -fsSL https://chatgpt.com/codex/install.sh | sh
Cursor Agentcursor-agentcurl -fsS https://cursor.com/install | bash
OpenCodeopencodecurl -fsSL https://opencode.ai/install | bash
Pipicurl -fsSL https://pi.dev/install.sh | sh

Log into the CLI directly first (usually /login inside it) so Squid can spawn it with working credentials.

2. Install and run AgentSquid

curl -L https://github.com/agent-squid/squid/archive/refs/tags/v0.1.tar.gz | tar xz
./squid-0.1/bin/start.sh

3. Open the browser UI

http://127.0.0.1:8000

4. Create your first agent

Agents are created in the UI. An agent is a named config:

agent name (@agent) = harness (Claude Code) + provider (Claude) + model (claude-sonnet-5) + working directory

Once an agent exists, start typing anywhere in the chat: #launch@claude write the release notes. The topic (#launch) and agent (@claude) are created the moment you use them — no separate setup step.
