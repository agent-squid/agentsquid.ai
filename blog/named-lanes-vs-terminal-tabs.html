Named lanes beat terminal tabs

Before Squid, running more than one coding agent meant more than one terminal window, each one identified only by whatever you happened to cd into or scroll back far enough to recognize. That works until you have three agents on two topics, step away for an hour, and come back needing to find the auth bug thread versus the release-notes thread versus the one you forgot you started.

Squid replaces that with a small addressing scheme: #topic@agent. A topic is a durable name for a workstream. An agent is a named config — harness, provider, model, and working directory. Combine them and you get a lane that persists across restarts, context clears, and reboots.

What that buys you

  • #bug@cursor always resumes the same native session, so context isn't rebuilt from scratch.
  • #launch@codex! runs a parallel adhoc turn without touching the main session's history.
  • #ops@opencode!3 runs adhoc with only the last three exchanges, when full history would be noise.
  • Every lane is filterable and resumable from the browser UI, and from a phone over Tailscale.

The underlying CLI still owns the real conversation. Squid isn't reimplementing agent memory — it's giving the sessions you already have names you can actually use, and a way to see them all in one place instead of a taskbar full of terminal icons.
