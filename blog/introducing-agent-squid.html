Introducing Agent Squid

Local coding agents got good fast. Claude Code, Codex, Cursor Agent, OpenCode, and Pi can all carry a real session, hold context, and get work done in a terminal. The problem was never any single agent — it was running several of them at once, remembering which terminal tab had which conversation, and losing track of how many tokens you'd actually burned getting there.

Agent Squid is a meta harness that sits in front of the CLIs you already use. It doesn't replace them or proxy their model calls — it wraps the native session, gives it a durable name, and puts a browser UI around the whole workflow: queues, history, live process control, mobile access, and usage analytics.

Every token counts

That's the line on the homepage, and it's meant literally. Squid tracks tokens, cache reads and writes, reasoning tokens, cost, and duration per topic and per agent, so "how much did that session cost" stops being a guess.

Named lanes, not tabs

Instead of unlabeled terminal windows, work lives under #topic@agent. Type #launch@claude to keep working a session, add ! for a parallel adhoc turn, or resume from your phone over Tailscale. We wrote more about why that model beats a desktop full of terminal tabs in the next post.
