Workflow

Turn terminal sprawl into named workstreams.

Topics and agents are lightweight enough to create as you type, but durable enough to recover after a reboot, filter later, or compare against adhoc runs.

Core features

The missing operating layer for coding agents.

Squid does not replace your agents. It gives them shared controls, history, context management, and visibility while they keep running locally.

Native sessions

Real CLI behavior

The agent owns the conversation context. Squid tracks session handles, topics, history, and UI state.

Pins

Context you can move

Pin or bookmark useful answers and inject them into another session or fresh adhoc prompt.

Remote

Couch coding

Use Tailscale-backed private access so your local machine keeps working while your phone controls the UI.

Queues

Ordered where it matters

Session work queues per lane, while independent adhoc prompts can run immediately.

Analytics

Every token counts

Inspect per-prompt usage and roll up cost by time, topic, backend, or agent.

Recovery

Find old work again

Filter history by topic and agent instead of searching shell scrollback or resume pickers.

Architecture

Local first, browser controlled.

The server binds locally, stores operational state in SQLite, and talks to local coding-agent subprocesses. The browser is the control surface, not the runtime.

Meta harness

Where Squid sits

Squid sits between your browser and local CLI agents. It provides routing, queueing, process ownership, analytics, and context workflows while your machine remains the execution environment.
