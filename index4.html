Mental model

Squid is a layer around real CLI sessions.

The coding agent still does the work on your local machine. Squid makes those sessions visible, recoverable, filterable, measurable, and controllable from a single browser UI.

01

Not another chat app

Squid does not send prompts to a generic model chat backend. It coordinates local coding-agent CLIs.

02

Not the coding engine

Claude Code, Codex, Cursor Agent, OpenCode, and Pi remain the tools doing the work.

03

The operating layer

Squid adds topic lanes, queues, resume handles, context pins, stop controls, history, and usage metrics.

Command model

Tags become your agent workspace.

Create a topic the moment you type it. Use a sticky agent when you want continuity, or add ! for a fresh adhoc run.

Examples

One thread, many lanes

Use session lanes for ordered work. Use adhoc lanes for comparison, short reviews, or limited-context checks. The UI keeps each workstream recoverable by topic and agent. 

# session lane
#launch@claude write release notes

# parallel adhoc review
#launch@codex! review the diff

# limited context comparison
#bug@cursor!3 reproduce the failure

Feature set

Built for the point where terminal tabs stop scaling.

Use Squid when you need several agents running, when sessions need names, when context needs to move, and when usage needs to be visible.

COUNT

Token analytics

Inspect usage per prompt and roll it up by time, topic, backend, or agent.

UNIFY

One browser UI

Run multiple local coding agents from a single interface instead of scattered terminals.

SHARE

Context pins

Pin useful answers and inject them into another lane or adhoc prompt.

CLEAR

Controlled resets

Use clear and compaction flows when long-running sessions grow stale.

PARALLEL

Independent jobs

Run independent work immediately while ordered session work queues per lane.

REMOTE

Phone access

Use private Tailscale access so your workstation keeps coding while your phone controls Squid.

Quick start

Install a backend CLI, then start Squid.

The detailed setup lives in GitHub. The short version is: install and log into at least one supported agent CLI, start Squid, then open the local browser UI.
