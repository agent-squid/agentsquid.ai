Quick start

Install one backend CLI. Start Squid. Open the UI.

AgentSquid works with supported local agent CLIs. Install and log into at least one backend first, then start Squid locally at 127.0.0.1:8000.

Basic usage

Tags are the UI for agent work.

Create topics as you type, switch agents with @agent, and use ! when you want an immediate one-off run with limited or no session history.

#topic@agent message Continue that topic and agent session.
#topic@agent! message Run a parallel adhoc turn with no session history.
#topic@agent!3 message Run adhoc with only the last three exchanges as context.
/stop Stop the current process scope.
/stopall Stop and drain the current topic.
/clear Create a new session when old context gets stale.
/remote Show a QR code for private phone or tablet access via Tailscale.

Squid Flow

Chain agents together with one line of syntax.

A route isn't limited to one agent. Add a hop operator and Squid dispatches the handoff, the round trip, or the join server-side — no open browser tab required. Read the full Squid Flow reference for the grammar and execution model.

#squid@codex>@review! One-way handoff: codex's output becomes fresh @review's prompt.
#t1@a1<>#t2+#t3 Round trip with a joined target: @t2 and @t3 run in parallel, then one return turn pins both.
#t1@a+#t2@b>#t3 Origin join: #t3 runs once, after both origins finish, with both pinned into its turn.
#squid@codex=3:1d>@review Scheduled: three independent one-way handoffs to @review, one per day.

Why it matters

Squid controls the workflow, not the model.

Most chat UIs send messages to a model API and render text back. Squid wraps real local coding-agent CLIs, preserves their native session behavior, and adds the missing operating layer around them.

Use the agent directly when one terminal is enough. Use Squid when you need named sessions, mobile access, process control, recoverable long-running work, and usage analytics.

Good fit

Built for multi-agent local workflows.

Squid is narrow by design: an orchestration layer for local coding agents, not a hosted team chat product or another coding engine.
